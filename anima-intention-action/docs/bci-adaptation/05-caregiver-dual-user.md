# 05 · Caregiver as Second User

> Healthcare reality: the patient is one user, but not the only user. The caregiver (family member or professional aide) is a co-equal stakeholder whose needs the product must serve.

## Why this is in the framework, not just the UI

Most assistive-tech products treat the caregiver as an afterthought ("we'll add a parent app later"). This adaptation puts the caregiver into the framework itself:

- TaskSpec carries `caregiver_consent_required` and `caregiver_present` flags
- Five-factor assessment (PEA) writes caregiver-readable summaries
- Test-and-Check Gate 5 can require caregiver presence for high-risk skills
- The Safety Watchdog gives the caregiver an unconditional override channel

## Two user roles, two views

| Role | Primary goal | Bandwidth | Interface style |
|---|---|---|---|
| **Patient** | Accomplish ADL with dignity | very low (5-8 bits/s via BCI) | minimal, two-stage confirmation, large clear options |
| **Caregiver** | Monitor, intervene, report | normal (touch / voice / keyboard) | dashboard, override controls, reports |

## Caregiver's interaction surfaces

### 1. Live observation panel
- Current intent token + confidence
- Five-factor circular gauges
- Active subtask in the behavior tree (highlighted)
- Active embodied device + its capability
- Recent E-stop / failure events

### 2. Intervention controls
- **Pause** (soft halt; current subtask completes then waits)
- **Take over** (hard halt; caregiver gets touch / voice control of device)
- **Adjust** (raise / lower confidence threshold for next N tasks)
- **Skill whitelist** (disable specific skills temporarily, e.g., near-face skills until calibration)

### 3. Daily summary (PEA-driven)
- Generated by Sonnet 4.6 at end of day from PEA entries
- Format: one paragraph (80-120 words) + key metrics
- Example:
  > "Today Mr. Li attempted 23 ADL tasks. 17 completed independently (74%), 4 needed your assistance (mostly for cup retrieval after the 3pm calibration drift), and 2 were aborted by him. Fatigue rose modestly between 4-5pm — consider a longer break before the 7pm meal block. The right-arm reaching skills had a 12% lower success rate than yesterday; suggest a 5-minute recalibration tomorrow morning."

### 4. Configuration (limited)
- Confidence threshold for skipping 2nd confirmation
- 2nd-confirmation timeout (1-5s)
- Caregiver paging method (vibration / sound / silent)
- Skill whitelist defaults

The caregiver **cannot** modify:
- Force envelopes
- E-stop channels
- Test-and-Check gate logic
- Audit trail

## TaskSpec fields the caregiver controls

```ts
{
  constraints: {
    user_in_loop: boolean,
    caregiver_consent_required: boolean,    // ← caregiver-set per skill
    abort_on_low_confidence: number,         // ← caregiver threshold
  },
  fallback: [
    "request_second_confirmation",
    "downgrade_to_caregiver_assist",
    "notify_caregiver"                       // ← always true; cannot be disabled
  ]
}
```

## Communication patterns

### Patient → Caregiver
- Explicit `CALL_HELP` intent (system-level skill)
- Implicit: any task that triggers `notify_caregiver` fallback
- Implicit: any sustained low-confidence period (caregiver gets a soft alert)

### Caregiver → Patient
- Voice (caregiver speaks; patient hears)
- "Acknowledge" button — caregiver lets patient know help is coming
- Override notice — patient sees "caregiver is taking over" before control transfers

## Demo screens

The demo opens the caregiver view at `/caregiver`. In the same browser session it stays state-synced with `/` via BroadcastChannel API.

For interview demos, caregiver and patient views can run side-by-side in a split layout — single-screen demonstration is fully supported (see `05-anima-intention-action-design.md` D9).

## Caregiver as data source

Caregiver feedback fields are first-class citizens in PEA:

```ts
type PeaEntry = {
  // ... other fields
  caregiver_feedback?: {
    rating: 1 | 2 | 3 | 4 | 5;
    note?: string;
    suggested_skill_swap?: string;
    flag_for_clinician?: boolean;
  }
}
```

These feedback entries are weighted heavily in the PEA retrieval (importance × 2.0 in the original three-factor formula). They are how the system learns user-specific preferences without requiring the patient to type.

## Out of scope

- Caregiver clinical training app (separate product)
- Caregiver scheduling / shift management (separate product)
- HIPAA / NMPA-grade EHR integration (downstream concern)
